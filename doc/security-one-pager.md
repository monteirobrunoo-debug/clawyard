# PartYard ClawYard — Security One-Pager

_For regulated / defence / enterprise procurement. Every claim below is
verifiable from the public repo; reference links are at the end._

## What the customer data actually does

When a ClawYard agent answers a prompt, the data path is:

```
Customer browser
    │  TLS 1.3 to clawyard.partyard.eu
    ▼
nginx on ClawYard VM  ──────────── (TLS terminated inside our VPC)
    │  loopback (never leaves the box)
    ▼
PHP-FPM (Laravel app)
    │  runs PiiRedactor — pass 1
    │  loopback
    ▼
FastAPI redactor on 127.0.0.1:8787
    │  runs PiiRedactor — pass 2  (belt-and-suspenders)
    │  TLS 1.3
    ▼
api.anthropic.com
```

**What Anthropic receives**: the prompt after both redaction passes. PII
classes we cover (see next section) are replaced with deterministic
tokens (`[EMAIL_REDACTED]`, `[NIF_REDACTED]`, `[IBAN_REDACTED]`, etc.)
before the bytes ever leave our infrastructure.

**What Anthropic stores**: Anthropic's Commercial Terms apply —
no training on inputs, 30-day retention for operational purposes,
deletion on request. ClawYard is billed through an Anthropic
commercial API key, not a consumer account.

**What PartYard stores about the call**: two JSONL log streams on the
Digital Ocean droplet — one per call with metadata only (request id,
model, token count, duration, status), one with redaction counts only
(how many of each PII category fired). **The prompt text itself is
never written to disk.** 90-day rolling retention, then compressed and
deleted by logrotate.

## PII categories covered today

Measured against the reproducible audit in
`llm-proxy/eval/REPORT.md` (run `python -m eval.run_audit` to regenerate):

| Category | Recall on test set | Notes |
|---|---:|---|
| Email addresses | 100% | Local part masked, domain preserved for context |
| Portuguese NIF / NIPC | 100% | Mod-11 check; SAP doc numbers preserved |
| IBAN (any country) | 100% | Pattern + length check |
| Portuguese Cartão de Cidadão | 100% | 8-1-2-1 format |
| Credit card numbers | 100% | Luhn-validated; random digit runs preserved |
| Secrets in `key=value` form | 100% | `password`, `token`, `apikey`, `bearer`, … |
| PEM private-key blocks | 100% | Matches both RSA and generic PKCS8 |
| International phone numbers | 66% on test set | Domestic PT numbers without `+` prefix are a known gap — see roadmap |

**Aggregate**: 90.5% recall on categories the redactor claims to cover,
0.0% false-positive rate on non-PII content (including 12 realistic SAP
export lines).

## PII categories NOT covered today — disclosed, not hidden

| Category | Gap | Remediation |
|---|---|---|
| Person names | Not redacted | Sprint 1: spaCy `pt_core_news_md` NER pass, ~50 ms overhead |
| Postal addresses | Not redacted | Sprint 1: regex keyed on `NNNN-NNN` postcode + street prefix |
| Dates of birth | Not redacted | Sprint 1: ISO-format regex + NER for written dates |
| Passport / docket numbers | Not redacted | Sprint 1: `[A-Z]{1,2}\d{6,8}` pattern |
| URLs with embedded basic-auth | Partial | Sprint 2: dedicated `user:pass@host` rule |

Each of these is <1 day of engineering including audit fixtures. The
reason they aren't already shipped is scope discipline, not capability.

## Infrastructure & encryption

- **Hosting**: Digital Ocean droplets, Frankfurt region (EU data
  residency). Managed through Laravel Forge.
- **TLS**: 1.2+ enforced at every hop. Public ingress
  (`clawyard.partyard.eu`, `llm-proxy.partyard.eu`) is TLS 1.3 by
  default. HTTP → HTTPS redirect enforced in nginx. Plain HTTP is
  permitted only on the `127.0.0.1` loopback between PHP-FPM and the
  FastAPI redactor — bytes never touch a physical NIC in cleartext.
- **Secrets**: API keys live in `.env` outside the webroot, read at
  boot, not checked into git. Rotation is a one-file edit + `service
  php-fpm reload`.
- **Outbound**: the only upstream ClawYard talks to (apart from the
  customer-facing mail relay) is `api.anthropic.com`. A firewall egress
  allow-list is available on request.

## Audit trail

- `/home/forge/llm-proxy/logs/access.jsonl` — one line per upstream call:
  request id, model, token count, duration, status, client IP. No
  prompt text.
- `/home/forge/llm-proxy/logs/redact.jsonl` — one line per call with
  counts per PII category (`{"email": 2, "nif": 1, …}`). No prompt
  text.
- `/var/log/nginx/llm-proxy.access.log` and `.error.log` — nginx-level.
- Git history of `llm-proxy/redactor.py` + `app/Support/PiiRedactor.php`
  is the change log for what the regex covers.
- `llm-proxy/eval/REPORT.md` is regenerated by CI and committed, so
  recall/FP numbers are diff-able across releases.

## What we explicitly do NOT claim

- We do **not** claim the prompt is invisible to Anthropic — the
  redacted prompt text still reaches their API under their Commercial
  Terms. What we claim is that the PII classes listed above are gone
  before it does.
- We do **not** claim 100% recall. Recall on the test set is 90.5% on
  covered categories; the full number and the test corpus are in the
  repo.
- We do **not** run a model ourselves today. A hybrid routing layer
  that sends high-sensitivity prompts to a locally-hosted model is on
  the roadmap (see `doc/hybrid-model.md`) and does not exist in
  production yet.
- We do **not** have formal SOC 2 / ISO 27001 certification. The
  architecture maps cleanly to both control sets; we can provide
  per-control evidence on request.

## Roadmap the customer can hold us to

1. **Named-entity redaction** (names + organisations) — closes ~40%
   of current gap. Target: next release cycle.
2. **PT postal address & DOB rules** — closes another ~25%. Target:
   same release.
3. **VM separation** — move the redactor to a dedicated Digital Ocean
   droplet so a compromise of the application tier does not imply a
   compromise of the audit trail. Shared-secret auth between tiers.
   Design in `doc/vm-separation.md`. Target: 2 weeks after customer
   sign-off.
4. **Hybrid model routing** — local Llama-class model for
   high-sensitivity prompts, Claude for the rest. Classifier in
   `app/Support/SensitivityClassifier.php`. Design in
   `doc/hybrid-model.md`. Target: 6 weeks, dependent on hardware
   sizing.
